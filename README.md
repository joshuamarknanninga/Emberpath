# EMBERPATH MVP

EMBERPATH is a production-shaped survival knowledge platform with structured entries, cross-references, cautious AI guidance, offline exports, map reporting, lightweight social features, and JWT auth.

## Product Summary

- **Knowledge Library** with original entries, safety notes, source/provenance metadata, filtering/search, and save-to-profile support.
- **Cross-reference system** combining explicit relations and shared-tag suggestions.
- **AI Guide** route at `/api/ai/advice` that provides structured safety-first responses with uncertainty and fallback references.
- **Offline pack builder** for JSON and static HTML export (flash-drive friendly), plus PWA service worker caching.
- **Gamified map layer** (Leaflet) for resources/hazards/meetups and points for reports.
- **Lightweight social** feed with posts, comments, and messaging scaffold.
- **JWT auth** for register/login/profile with saved entries and progress.

## Tech Stack

### Frontend
- React (JavaScript) + Vite
- Tailwind CSS
- Framer Motion
- React Router
- React Leaflet

### Backend
- Node.js + Express (JavaScript)
- MongoDB + Mongoose
- JWT authentication (`jsonwebtoken`)
- `bcryptjs` for password hashing

## Setup Instructions

1. Copy env:
   - `cp server/.env.example server/.env`
2. Install dependencies:
   - `npm run install:all`
3. Ensure MongoDB is running locally (default: `mongodb://127.0.0.1:27017/emberpath` when `MONGO_URI` is not set).
4. Seed data:
   - `npm --prefix server run seed`
5. Run both server and Vite client:
   - `npm run dev`
   - This script auto-installs dependencies if missing and auto-selects an available API port (starting from `5000`).
   - It injects `VITE_API_BASE` and `VITE_WS_URL` for the client automatically.
6. Open `http://localhost:5173`.

### Realtime WebSocket
- WebSocket endpoint: `ws://localhost:<selected-port>/ws` (the dev launcher auto-selects and injects this).
- The dashboard subscribes to live system events (map/social updates).

### Vite dev host note
- Vite is bound to `127.0.0.1:5173` with strict port mode to keep HMR websocket behavior stable in local development.

## API Highlights

- Auth: `POST /api/auth/register`, `POST /api/auth/login`, `GET /api/auth/profile`
- Knowledge: `GET /api/knowledge`, `GET /api/knowledge/:id`, `POST /api/knowledge/:id/save`
- AI: `POST /api/ai/advice`
- Offline: `POST /api/offline/packs`, `GET /api/offline/packs/:id/export`
- Map: `GET /api/map/reports`, `POST /api/map/reports`
- Social: `GET /api/social/feed`, `POST /api/social/posts`, `POST /api/social/posts/:postId/comments`, `POST /api/social/messages`
